# Phase 1 — Core module + prompt + config

**Part of:** [PLAN.md](PLAN.md) · **Status:** ✅ (green-lit 2026-06-11)

## Goal
Role profiles can be generated by the AI, saved to disk, reused from cache, and rendered as a text block — without touching the pipeline yet.

## Changes
- New `src/role-profile.js` — keying (`<role-slug>--<seniority-slug>`), cache-first `ensureRoleProfile`, null-safe `loadRoleProfile`, `renderRoleProfileBlock` with per-stage slices and the bi-weekly/feels-off competency filter.
- New `prompts/generate-role-profile.md` — generation prompt. Inputs are ONLY role title + seniority (never name, never notes — the file is shared across people).
- `config/models.json` + `src/models.js` — new `role_profile` stage model (gpt-5.4).
- New `scripts/test-role-profile.js` — offline checks, no API spend.
- New cache dir `data/role-profiles/`.

## Not in this phase
- No stage prompt changes, no pipeline injection (Phase 2).
- No CLI/web intake wiring (Phase 3).
- No trust gates or smoke scenario (Phase 4).

## Done when
- [ ] `node scripts/test-role-profile.js` passes (slug edge cases, corrupt-file safety, arc filtering, fallback text).
- [ ] One real profile generated and saved; rerunning is an instant cache hit (no API call).
- [ ] Product owner has tested the scenarios below and said go.

## Test scenarios — for the product owner
Walk through these yourself. Next phase waits for your green light.

1. **Generate a profile** — in a terminal in the project folder run:
   `node scripts/role-profile-demo.js "Staff Site Reliability Engineer" "Senior"`
   You should see "generated" plus a readable summary of the profile: what the job is about, known challenges, question themes, and the terms that role uses. ❌ Not OK if it errors, or the content reads generic (could apply to any job).
2. **It's saved** — open `data/role-profiles/staff-site-reliability-engineer--senior.json` in a text editor. You should see the same content as JSON, with the role title and seniority recorded. ❌ Not OK if the file is missing or contains anyone's name or your notes.
3. **Reuse is instant and free** — run the same command again. You should see "cached" and the answer appears immediately (no AI wait). ❌ Not OK if it says "generated" again.
4. **A different job is different** — run it once with a clearly different title, e.g. `node scripts/role-profile-demo.js "Junior Graphic Designer" "Junior"`. The two profiles should read clearly different — different challenges, different vocabulary. ❌ Not OK if they look like copies.
5. **Offline checks** — run `node scripts/test-role-profile.js`. You should see "All checks passed."
