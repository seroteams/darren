# Audit findings — UI look-and-feel initiative

*Generated Fri 17 Jul 2026 by a 9-agent audit (1 design-language extraction, 7 dimension audits, 1 synthesis). This file is the durable reference for the build phases — the synthesis became plan.md; this holds the underlying detail.*

## The target design language (extracted from the artifact Carl loved)

**North star:** The artifact is Sero's existing "quiet debrief" given dimension and ceremony. It keeps the identical light palette (same ink #1f2a37, paper #f5fafd, one sky accent #5aa9e6, Bricolage-over-Inter) but changes three things at once: (1) the briefing is framed like a made object — the tinted page colour becomes the ground INSIDE a rounded (18px), hairline-bordered, softly-lifted frame, with white cards floating on it, inverting the usual card-on-page scheme; (2) hierarchy is carried by a strict two-tier uppercase label system (accent-dark eyebrows, dim slot labels) beneath which the words the manager will actually say are set in the display face — content, not chrome, gets the Bricolage moments; (3) meaning is carried by tiny accent-coloured geometry instead of icons or boxes — 6px dot list markers, 3-dot confidence meters inside pills, a node-and-spine timeline, 30px numbered medallions. All of it is defined twice at the token layer, so the whole look holds in a hue-biased dark theme where darks stay on the brand's blue axis and are never grey or black.

### Principles
- Same palette, more dimension: tone-on-tone grounds plus ONE ink-tinted lift shadow replace flat borders-only depth — but only at the frame level, cards inside stay flat.
- Content is the hero: the sentence the manager will say gets display type (Bricolage 600, balanced wrapping); chrome gets 14px uppercase labels. Never the reverse.
- Meaning by motif, not icon: 6-7px accent dots, dot-meters, spines, and medallions do the signalling; icons appear only inside the semantic panels.
- Every tint is a triad: soft ground + hairline line-border + dark text of the SAME hue — never a bare tint or a grey border on a coloured panel.
- Hairlines carry structure, whitespace carries rhythm: 1px rules divide, adjacent cells share rules, and the gaps stay generous (8/14/18/24/40px).
- Dark mode is built at the token layer only — components never know; dark grounds are hue-biased toward the brand blue, and 'dark accent text' flips polarity to a pale tint so it stays readable.
- Motion is opt-in: transitions exist only under prefers-reduced-motion: no-preference, 150ms colour/border eases, nothing moves position.
- Reading measures are capped per role (34-64ch) inside a 900px container — narrower and calmer than a dashboard grid.

### Signature treatments (exact values)

**Frame / screen treatment (.screen)**  
Page-tint ground (var(--page) #f5fafd) INSIDE the component: 1px var(--border) hairline, border-radius 18px (--r-xl), padding 22px (30px at >=640px), box-shadow var(--shadow-lift). White cards then sit ON this tinted ground — the inversion is the signature.  
`.screen { background: var(--page); border: 1px solid var(--border); border-radius: var(--r-xl); padding: 22px; box-shadow: var(--shadow-lift); }`

**Lift shadow language (--shadow-lift)**  
One two-layer, ink-tinted, very-low-alpha shadow used ONLY on the frame: light = 0 1px 2px rgba(31,42,55,.04), 0 8px 24px rgba(31,42,55,.06) (shadow colour IS the ink hue, not black); dark = 0 1px 2px rgba(0,0,0,.3), 0 10px 30px rgba(0,0,0,.35). Tight contact layer + long soft layer; nothing else in the artifact has a shadow.  
`:root --shadow-lift (lines 47, 73, 84, 93)`

**Eyebrow (section-level label)**  
14px / weight 600 / letter-spacing 0.08em / uppercase / colour var(--accent-dark) #1b5d91 — the accent-dark colour is what lifts it above a plain caption. Optional leading number in ink-dim with font-variant-numeric: tabular-nums.  
`.eyebrow + .dir__num`

**Slot label (.lbl) — the second label tier**  
14px / 600 / letter-spacing 0.06em / uppercase / var(--ink-dim), display:block with 6-9px margin below. Deliberately quieter than the eyebrow (dim vs accent-dark, 0.06em vs 0.08em) — a two-tier system: accent-dark = section, dim = slot.  
`.lbl`

**Pills & scenario chips**  
border-radius 999px, padding 5px 11px, 14px / 500, tint-pair colouring: accent-soft bg + accent-dark text (scenario), gold-soft #fff4d9 bg + gold-text #6b4a00 (status), or 'plain' variant surface + 1px border + ink-dim. The status pill carries a 7px currentColor dot (b element) as a leading motif.  
`.mock-pill, .scenario span, .scenario span.plain`

**Confidence dot-meter chip (.conf)**  
An accent-soft/accent-dark pill (5px 11px, 999px, 14px/500) containing three 6px circles at 3px gap, all background: currentColor — off dots at opacity .3, on dots opacity 1. A tiny data-viz inside a chip; on the navy header it recolours to rgba(255,255,255,.12) bg + navy-ink automatically via currentColor.  
`.conf .dots i / i.on; .d4__conf`

**Accent-dot list markers (.plist)**  
list-style none; display:grid gap 8px (9px in panels); li padding-left 20px; ::before = 6px accent-coloured circle at left 4px / top 10px. The brand accent does bullet duty — no discs, no icons, 16px ink text.  
`.plist li::before`

**Display headings (Bricolage)**  
Bricolage Grotesque 600 everywhere, always with negative tracking and text-wrap: balance. Page h1: clamp(30px, 5vw, 42px) / lh 1.08 / -0.02em. Section names: 28px / 1.15 / -0.01em. Identity lines: 20-22px / 1.2 / -0.01em.  
`.intro h1, .dir__name, .ident__name`

**Bricolage-for-content (the opener moment)**  
The manager's actual opening sentence is set in the display face: hero size clamp(24px, 4.5vw, 32px) / 1.18, or in-card 18-19px / 1.25, always 600 / -0.01em / text-wrap: balance. Display type marks what to SAY, not just page titles.  
`.d1__opener, .d3__say, .d5__opener, .d2__callout`

**Tinted borderless callout**  
background var(--accent-soft), border-radius 12px (--r-lg), padding 14px 16px, NO border — content set in Bricolage 600 18px / 1.25 / -0.01em in full ink. Same family as the d5 banner (accent-soft, 16px 18px padding, lbl in accent-dark).  
`.d2__callout, .d5__banner`

**Spine + node timeline motif**  
2px vertical rail in var(--border-strong) at left 11px with content indented 34px; each phase gets a 24px node: accent-soft fill + 2px accent border + 7px accent inner dot; the terminal phase inverts (solid accent fill, white inner dot). Phase labels are 14px / 700 / 0.08em uppercase accent-dark with a de-capitalised dim time-tag beside them.  
`.d2__spine::before, .d2__dot, .d2__phase--after .d2__dot, .d2__plabel/.d2__ptime`

**Numbered step medallions**  
30px circle, accent-soft bg, accent-dark numeral at 15px / 700 / font-variant-numeric: tabular-nums, display:grid place-items center; steps separated by 1px border-top hairlines with 16px vertical padding, not boxes.  
`.d5__num, .d5__step`

**Hairline-ruled cell grid (cheat-sheet)**  
A single card cut into cells by SHARED 1px rules: card padding 0 + overflow hidden; cells padding 18px 22px with border-bottom, odd cells border-right at >=600px; full-width cells span 1/-1. Structure from rules, zero gap — reads as an index card.  
`.d3 .card, .d3__band, .d3__cell, .d3__full`

**Committed navy header band**  
One committed colour block per card: background var(--navy) #1b3a52 (dark: #16324a), body text var(--navy-ink) #eaf3fb, eyebrow var(--navy-dim) #a9c6dd, opener pure #fff, chip on rgba(255,255,255,.12). A dedicated 3-token sub-palette so type hierarchy survives on the dark ground in BOTH themes.  
`.d4__head, --navy/--navy-ink/--navy-dim`

**Semantic tint-triad panels (listen/avoid)**  
Each hue is a triad — soft ground + line border + dark text: mint #e4f7f1 / #bfe9dd / #0c4b3c and coral #fdece9 / #f4c9c3 / #ac1608. Panel = soft bg + 1px line-colour border + 12px radius + 16px padding; headings 14px / 700 / 0.05em uppercase in the -text colour; list icons stroke the -text colour. The hue-matched hairline (-line) is the new token concept.  
`.d4__col--listen / --avoid; --mint-soft/-line/-text, --coral-soft/-line/-text`

**Buttons**  
15px / 600 / line-height 1, padding 10px 16px, radius 4px (--r-sm). Primary: accent bg + white, hover swaps bg to accent-dark. Ghost: surface + 1px border-strong + ink, hover shifts border-color to ink-dim (border darkens, bg stays). Focus-visible: 2px accent outline, offset 2px. Transitions (background/border-color/color .15s ease) exist ONLY inside @media (prefers-reduced-motion: no-preference).  
`.btn, .btn--primary, .btn--ghost, final @media block`

**Spacing rhythm**  
Sections: 40px top padding. Frame: 22px pad (30px >=640px). Grids: 14-18px gaps; lists 8-9px; label-to-content 6-9px. Hairline rules: 24px vertical margin. Footers: 22-24px top margin. Page: 96px bottom padding, 900px max-width container, body lh 1.55. Secondary text measure-capped per role: 34ch (theme), 40ch (banner), 62-64ch (intros).  
`.dir, .screen, .d1__grid/.d2__body, .d1__rule, .wrap, .container`

**Theme toggle (segmented pill)**  
Container: surface bg + 1px border + 999px radius + 3px padding; segments 14px / 500 dim, 6px 14px padding, 999px radius; active state (aria-pressed=true) = accent-soft bg + accent-dark text; sets data-theme on the root.  
`.toggle, .toggle button[aria-pressed="true"]`

### Theming construction

Pure token-level dual theme — components are written once against ~22 custom properties and never mention a mode. Four blocks in order: (1) :root holds the light values (identical hexes to the app's existing tokens); (2) @media (prefers-color-scheme: dark) :root overrides every token for OS-dark; (3) :root[data-theme=\"light\"] and (4) :root[data-theme=\"dark\"] duplicate both full sets so an explicit in-page toggle beats the media query in BOTH directions (the toggle just sets data-theme on <html>). How the dark values were chosen: darks are hue-biased toward the brand's ink-blue, never neutral — page #0f151b, surface #18212b, surface-2 #141c25, borders #2a3642 / #33414f all carry blue; ink flips to blue-white #e9f0f6, dim to #9fb1c0. The accent LIGHTENS one step (#5aa9e6 → #6fb4e8) so it reads on dark, and — the subtle move — --accent-dark (the text-on-tint / eyebrow colour) flips polarity to a pale tint #b7dcf6, so every 'dark accent text on soft tint' pairing survives untouched. Each semantic triad inverts as a unit: -text becomes light (#ff9284 / #74e2c6 / #e6c877), -soft becomes a dark hue-tinted ground (#2a1c1a / #13251f / #2a2413), -line a dark hue border (#4a2f2b / #274036); accent-soft becomes #1c2e3d. The shadow deepens and enlarges (rgba(0,0,0,.3)/.35, 10px/30px). Fonts are self-hosted variable files (InterVar + Bricolage, font-weight 100 900, font-display: swap), so weights render identically in both modes.

### Genuinely new (the delta)
- DUAL THEME — the big one. tokens.css is light-only; the artifact adds a complete dark token set (hue-biased blue-black darks: page #0f151b, surface #18212b, borders #2a3642/#33414f), the accent-dark polarity flip (#1b5d91 → pale #b7dcf6), inverted semantic triads, and the 4-block :root / prefers-color-scheme / data-theme override mechanism with an in-page toggle

- Accent-dot list markers — 6px accent circle ::before, 20px indent, 8px grid gap; app lists have no marker treatment
- Bricolage-for-content + text-wrap: balance — display face on the actual opener sentence (18-32px clamps, lh 1.18-1.25) and balance on every heading; neither appears anywhere in the app CSS
- The tinted borderless callout — accent-soft block, 12px radius, no border, display-type content (and its banner sibling)
- The spine + node timeline motif — 2px rail, 24px two-state dots (soft+ring vs solid+white), 34px indent, uppercase phase labels with de-capitalised time tags
- The committed navy header band with its own 3-token sub-palette (--navy / --navy-ink / --navy-dim) and white-alpha chips — a sanctioned 'one committed colour block' pattern (currently adjacent to DESIGN.md's no-dark-glass caution, so it needs an explicit rule)
- The semantic -line step — hue-matched hairline borders (#bfe9dd mint, #f4c9c3 coral) completing soft/line/text triads; tokens.css has no border step per hue, and the artifact's gold-text #6b4a00 is a new value between gold-800 and gold-900
- Numbered medallions (30px accent-soft circle, accent-dark 700 numeral) and font-variant-numeric: tabular-nums usage — tabular figures are untokenized in the app
- The hairline-ruled cell grid — one card cut by SHARED internal 1px rules (border-bottom + odd-cell border-right, zero gap); app cards divide by gaps, not shared rules
- Opt-in motion — transitions declared ONLY inside @media (prefers-reduced-motion: no-preference); the app honours reduced-motion but doesn't gate this direction
- Button size step 15px/600 with 10x16 padding and line-height 1 (app spec is 8x16), plus the ghost hover that shifts border-color to ink-dim
- The 900px reading container + section rhythm (40px section heads, 22/30px frame padding, 24px rule margins, 96px page tail) — a narrower, airier page grid than the app's 1024px --container-medium

## Dimension: SPACING

**Current state:** The app has a clean 8px spacing grid at the token layer (--sero-space-0 through -24 in admin/src/styles/design/tokens.css) and a good gap-based layout kit (.l-stack/.l-row/.l-grid/.l-cluster in design/layout.css) — but no role-level spacing tokens, so every component picks raw values: card padding alone renders at 24px (.card), 20px (.card-flat), 14x16px (.fb-item), 16px (.cl-phase-head), 16x18.4px (.meeting-card), and 20x24px (.prep-brief__row). Stage markup runs three competing rhythm systems side by side — .l-stack (110 uses), Tailwind space-y-* (77 uses), and mb-2/mb-3/mb-4 heading margins (intake.js, onepage.js) — while the older shared CSS (briefing.css, buttons-inputs.css, stage-extras.css) is dense with one-off rem margins (margin-top: 0.35/0.4/0.5rem) instead of gap. Against the artifact's explicit rhythm (40px sections, 22/30px frame, 8/14/18/24px gaps, 24px rule margins, 96px page tail, 900px container), the app is close on list gaps (8px) but cramped on row/list-item density in the briefing bottom half and has no defined section rhythm, page tail, or mid-size reading container.

**Scale:** Token fixes: 1 file (tokens.css). Shared-component fixes: ~8 files in admin/src/styles/design/ (cards, layout, briefing, buttons-inputs, stage-extras, admin-tables, base, one-page-run) — these cover both apps since frontend imports the same barrel. Full ad-hoc-margin/space-y sweep: ~42 admin stage files use spacing utilities (77 space-y occurrences, ~79 mt-/mb-/gap- occurrences across 21 files) plus ~6 frontend stages and 7 bolt-on CSS files outside the barrel. Realistic: token + shared-component work fixes ~70% of what a user sees in ~9 files; the long-tail markup sweep is 25-30 stage files.

## Dimension: Controls (buttons, inputs, chips/pills, badges, toggles, dropdowns, focus states)

**Current state:** The canonical layer is real and mostly on-pattern: `.btn` + ghost/danger variants at the 4px radius token, two sanctioned input variants (compact boxed `.apm-field__input` and the big borderless session `.input`), a strong global double-ring `:focus-visible` in base.css, and DESIGN.md §5 already specs the artifact's chip colour logic (tint bg + dark same-hue text, 999px). But execution has fragmented: roughly 15 bespoke pill/badge families across the two apps (`um-badge`, `pd-pill`, `el-pill`, `fb-pill`/`fb-verdict`, `lp-pill`, `arc-chip`, `pl-chip`, `fp-chip`, `cl-badge`, `brutal__badge`, `meeting-card__badge`, `lib-badge`, `cmp-verdict-tag`, `.pill`, `mcr-chip`/`mcr-owner`) each re-declaring size/padding/radius, three unrelated segmented-control builds, two parallel dropdown-menu builds, and the customer guided flow runs an entire off-system control kit (`mcr-btn` at 8px radius, 13–13.5px chip text, ~120 hardcoded hexes). The artifact's confidence dot-meter and dot-motif status pills have no equivalent anywhere — confidence renders as plain paragraphs in all seven briefing layouts.
