🗝️ Claude Cheat Sheet

One word does the whole job. Say it in any session — Claude knows the rest.

The magic words

Say…
checkpoint
Mid-work: saves everything — commit, STATUS refresh, chat-log.
Fresh session: brings the whole picture back — done / awaiting your QA / your move.
Use it before you walk away, and first thing when you come back.
Say…
tested, good
Closes the phase properly: plan, STATUS, board, badges, log, changelog — all moved together, then committed. Nothing drifts.
Only after YOU walked the test scenarios. ("green light" works too.)
Say…
night test
The full overnight QA sweep — free checks first, paid runs capped & tallied, report waiting in the morning. No fixes, no commits.
Last thing before bed. Add a budget line if you want a different cap.
Say…
commit my work
Commits ONLY this session's files. If other work is mixed in, you get an A/B/C choice — never a silent sweep. Push only if you say push.
Any time you want work saved to git.
Say…
review this run
Full run review with the scorecard (Understood / Filtered / Shown). It builds the context block itself — you never paste Report/Role/Meeting-type again. Just give the log folder.
After any pipeline run you want judged.
Say…
split this into phases
The Darren Method: plan folder + phase files, one phase at a time, you green-light each. ("darren method" works too.)
Any change too big for one sitting.
Say…
start the app
Dev servers up via the preview tools (API on 3001, admin on 3000) — no setup chat needed.
Whenever you want to click around the app.
Say…
where are we
Same as a checkpoint restore — the plain-words picture from STATUS.md, the active plan, and git. No re-explaining.
Anytime you feel lost between sessions or agents.

💷 The money rules (already standing — no need to repeat them)

Free checks always come first. A task may use one paid run (~$0.35) without asking — only if free can't prove it, always the smallest run. A second paid run needs your yes, every time. Full gate ≈ $3 — never run by default.

🤖 What now happens automatically (you never have to ask)

⚠️ Guardrails — every message you send is checked against the five drifts (goal, pace, honesty, money, scope); a warning leads the reply only if one fires.
📌 Reply format — slim title on top, one 🧭/🔵/🔴 box at the bottom of every reply.
✅ Task list — phased work shows a live tick-off checklist, plus STATUS.md stays current on every phase boundary.

House habits (one-liners)

🧪
"Done" means verified at the destination — the database was queried, the real page loaded — never "the code should do it".
🌳
Running two sessions at once? Ask the second one to "make me a worktree" — each session gets its own safe copy.
📄
Every file-edit reply ends with a files-edited table — you always see exactly what was touched.
🗒️
Ideas mid-phase go to "Parked" — they're saved, not built, so the current phase stays small.
