Sero × Flowbite

The Sero look in one page: Flowbite 2.5.2 components wearing our colours. New screens copy from here. Controls round at 4px, cards at 12px. Nothing smaller than 14px. One blue action per screen.

✔ Before you build

Check every new or touched screen against this list. If one fails, it's not done.

1. Colours come only from the tokens — never type a hex code in a screen file.

2. Nothing under 14px. No exceptions.

3. One blue action per screen. If two blue buttons compete, one is wrong.

4. Corners: 4px on controls, 12px on cards.

5. Every screen ships with its empty, loading, and error states designed.

6. Layout uses the shared primitives (.l-stack, .l-grid…) — no bespoke scaffolding.

7. Anything destructive goes through the confirm dialog.

8. Works at phone width — no sideways scroll, everything tappable.

9. One date format everywhere: Mon 18 Nov 2024.

10. Plain words. Focus ring on everything interactive. No nested cards, no side-stripes, no gradient text.

📋 What Sero needs

Every component type Sero uses, from the Figma demo board and the live app. ✅ = on this sheet already · ⬜ = still to add. Cross off what we don't need; shout if something's missing.

  • ✅ Avatars + count badge
  • ✅ Inputs (compact boxed + big session variant, error state)
  • ⬜ Checkbox · radio · toggle switch
  • ⬜ Search box (users, teams)
  • ⬜ Date + time picker (schedule a 1:1)

Layout & navigation

  • ✅ Topbar + icon rail
  • ✅ Cards (content + request)
  • ✅ Side panel / slide-over
  • ⬜ Tabs (the 1:1 stages: Preparing · Catchup · Requests · Rating · Feedback · Goals · Summary)

  • ⬜ Page header (title + actions row)
  • ⬜ Welcome banner (dashboard hero)

Data & content

  • ✅ Table (one canonical style — user lists, run lists)
  • ⬜ Score elements: score pill ✅ · star rating · the axis slider (Sero's signature)
  • ⬜ Goal columns (To do / Doing / Done)
  • ⬜ Chart card (team scores, engagement lines)
  • ⬜ Legend / key row (request status colours)

Feedback & states

  • ✅ Modal · dropdown menu
  • ✅ Toast / inline alert (saved ✓, error, warning)
  • ⬜ Tooltip
  • ⬜ Empty state (no requests yet, no team yet)
  • ⬜ Loading skeleton + thinking state
  • ⬜ Confirmation dialog (delete / irreversible)
  • ⬜ Login / signup card
